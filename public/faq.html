Frequently Asked Questions

ReverseQR

General Questions

What even is this?

ReverseQR is a secure, end-to-end encrypted file and message sharing service. It allows you to securely share files and text with others without creating accounts or storing data on our servers long-term.

How does the encryption work?

ReverseQR uses ECDH (Elliptic Curve Diffie-Hellman) for secure key exchange and AES-256-GCM for encrypting your messages and files. Your encryption keys never leave your browser—the server cannot decrypt your content. However, a mailicious server operator could send a different webpage that do not encrypt the data, so only trust this service as much as its host.

Is my data really private?

Yes. All messages and files are encrypted in your browser before being sent to our server. Only the intended recipient's browser can decrypt them. We never have access to your encryption keys or plaintext content.

Getting Started

How do I send a file?

1. Click "Send" on the home page
2. You'll receive a 9-character code and a QR code
3. Share the code or QR code with your recipient
4. Once they connect, select files to send
5. Files are encrypted and sent over the secure connection

How do I receive files?

1. Click "Receive" on the home page
2. Enter the 9-character code you received from the sender
3. Verify the Security Fingerprint matches what the sender sees
4. Wait for the sender to share files
5. Click download to receive the encrypted files

What's the Security Fingerprint?

The Security Fingerprint is a 3-word combination that both sender and receiver see after connecting. You should verify these words match through a separate communication channel (call, text, in-person) to confirm you're connected to the right person and not being intercepted.

Sessions & Data

How long does my session last?

Sessions automatically expire after 15 minutes of inactivity. This helps protect your privacy by limiting the time your session remains active.

How long are files stored?

Files are automatically deleted after 30 minutes, regardless of whether they were downloaded. This ensures your shared data doesn't persist on our servers.

What happens if the recipient doesn't connect in time?

If the recipient doesn't join the session before it expires, the shared files and messages will be deleted. The sender will need to create a new session and share a new code.

Technical Questions

What file types can I share?

You can share any file type—documents, images, videos, archives, etc. There are no restrictions on file types, only on file size and total transfer limits.

What's the maximum file size?

File size limits depend on your browser's capabilities and available memory. Most modern browsers can handle files up to several gigabytes, but practical limits may be lower on mobile devices.

Does ReverseQR require an account?

No. ReverseQR is completely anonymous—no accounts, no registration, no personal information required.

Can I use ReverseQR on my phone?

Yes. ReverseQR works on any device with a modern web browser, including smartphones and tablets. The QR code feature is especially useful for mobile-to-desktop sharing.

Security & Privacy

Can the server admin see my files?

No. All encryption and decryption happens in your browser. The server only stores encrypted data and has no access to your encryption keys. Even the server administrator cannot read your files.

Is ReverseQR secure over public WiFi?

ReverseQR uses HTTPS encryption for the connection between your browser and our server, and all content is additionally encrypted with AES-256-GCM. This provides strong protection even on public WiFi.

Who can access my shared content?

Only the person who has your session code or QR code can connect to your session. Once connected, both parties can see all shared messages and files. After the session expires, all data is deleted.

What data do you log?

We log IP addresses and timestamps for security and abuse prevention, but we do not log any message content or file data. All logs are retained for operational purposes only.

Troubleshooting

The connection keeps dropping

This could be due to network issues or browser WebSocket support. Try refreshing the page and reconnecting. Make sure you're using a modern browser (Chrome, Firefox, Safari, Edge) with WebSocket support enabled.

I can't scan the QR code

Make sure your device has good lighting and the camera is in focus. If scanning still doesn't work, you can manually enter the 9-character code instead. Both methods work equally well.

The Security Fingerprint doesn't match

If the 3-word fingerprint doesn't match between sender and receiver, there may be a connection or security issue. Close the session and start over with a new code. If this keeps happening, there may be a network problem or potential security issue.

Files won't download

Make sure the sender's session hasn't expired. If you're downloading large files, the process may take a while. Check that you have enough disk space and try downloading in a private/incognito window if you're having issues.
